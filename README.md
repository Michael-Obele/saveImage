# Save Image

A tiny browser extension that lets you click any image on a page to download it. Built with [WXT](https://wxt.dev/) + [Svelte 5](https://svelte.dev/).

The extension injects a crosshair picker overlay on the active tab; hovering an image outlines it in green, clicking it triggers a download via `chrome.downloads`.

> **Status:** Not published to any store. Built artifacts are distributed via GitHub Actions and installed manually (see [Installation](#installation)).

## Features

- Crosshair-style image picker injected on every page
- One-click download through the browser's native download manager (`saveAs: true`)
- Cross-browser: Chromium-based browsers and Firefox
- MV3 manifest, generated by WXT

## Project Structure

```
src/
  entrypoints/
    background.ts        ← Service worker: handles download messages
    content.ts           ← Content script: picker overlay + click handling
    popup/               ← Toolbar popup UI (Svelte 5)
  assets/                ← Static assets bundled by WXT
wxt.config.ts            ← WXT manifest + build config
```

## Development

Prerequisites: [Bun](https://bun.sh/) (or Node 20+ with npm/pnpm).

```bash
bun install
bun run dev              # Chrome dev with HMR
bun run dev:firefox      # Firefox dev with HMR
bun run check            # svelte-check / TypeScript
```

## Building Locally

```bash
bun run build            # → .output/chrome-mv3/
bun run build:firefox    # → .output/firefox-mv3/
bun run zip              # → .output/save-image-0.0.0-chrome.zip
bun run zip:firefox      # → .output/save-image-0.0.0-firefox.zip + sources zip
```

The unzipped output in `.output/{chrome|firefox}-mv3/` is what you load as an unpacked extension.

## Releases

Every push to `main` creates a [GitHub Release](../../releases) with both the Chrome and Firefox builds attached. Each release gets an auto-incrementing version (`v0.0.1`, `v0.0.2`, …). Tagged releases (`v1.0.0`, …) work the same way with the tag name as the version.

You can also trigger a build manually from the [Actions tab](../../actions/workflows/build.yml).

## Installation

This extension is **not on the Chrome Web Store or Firefox Add-ons (AMO)**, so installation is manual. Follow the steps for your browser.

### Chrome / Chromium-based browsers (Edge, Brave, Arc, …)

Works on any Chromium browser with extension support.

1. Go to the [Releases page](../../releases) and pick the version you want.
2. Under **Assets**, download `save-image-chrome-*.zip`.
3. Unzip it — you'll get a folder containing `manifest.json`.
4. Open `chrome://extensions` in a new tab.
5. Toggle **Developer mode** on (top-right corner).
6. Click **Load unpacked** and select the unzipped folder.
7. Pin the extension from the puzzle menu for easy access.

**Updating:** Download a newer release, then on `chrome://extensions` click the **reload** icon on the extension's card. You may need to refresh open tabs so the content scripts reload.

**Notes:**

- The extension needs the `downloads` permission. Chrome will warn you on first install; this is normal.

### Firefox

Firefox has two install paths. **Use temporary add-ons for development and testing.** For a permanent install you need either AMO signing or a special Firefox build.

#### Option A — Temporary add-on (recommended for personal use)

The simplest path. The extension stays installed until you restart Firefox, then you reload it.

1. Go to the [Releases page](../../releases) and pick the version you want.
2. Under **Assets**, download `save-image-firefox-*.zip`.
3. Unzip it — you'll get a folder containing `manifest.json`.
4. Open `about:debugging` in a new tab.
5. Click **This Firefox** in the left sidebar.
6. Click **Load Temporary Add-on…**.
7. In the file picker, navigate into the unzipped folder and select `manifest.json`.
8. The extension appears under **Temporary Extensions** and works immediately.

**Updating:** On the same `about:debugging` → This Firefox page, click the **Reload** button next to the extension. You may also need to refresh open tabs so the content script reloads.

**Caveats:**

- Temporary add-ons are removed when Firefox restarts — you have to re-load after every restart.
- Some Firefox-only quirks (e.g. permission prompts) behave differently from signed installs. Functionality is otherwise identical.

#### Option B — Permanent install (Firefox Developer / Nightly)

1. Install [Firefox Developer Edition](https://www.mozilla.org/en-US/firefox/developer/) or [Firefox Nightly](https://www.mozilla.org/en-US/firefox/channel/desktop/#nightly).
2. Open `about:config`, accept the risk warning, and set:
   ```
   xpinstall.signatures.required = false
   ```
3. Download the `save-image-firefox-*.zip` from the [Releases page](../../releases) and unzip it.
4. Open `about:addons`, click the gear icon → **Install Add-on From File…** and pick the folder (or re-zip it first and pick the `.zip`).

**Updating:** Replace the folder on disk, then toggle the extension off and on in `about:addons` to pick up the new files.

**Caveats:**

- This setting disables signature enforcement for **all** add-ons in that Firefox profile — only do it in a dedicated dev profile.
- Do **not** use this profile for daily browsing; it is meaningfully less secure.

## Permissions

The manifest declares:

- `downloads` — to save the clicked image to disk
- `activeTab` — to send a message to the active tab's content script when you click the toolbar icon
- `<all_urls>` host permission — so the content script can run on any page you want to grab images from

See [`wxt.config.ts`](wxt.config.ts) for the full manifest config.
